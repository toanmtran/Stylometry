\section{Target of Data Collection}
The objective of this phase is to build a dataset of long-form articles ($\ge 1,500$ words) from the LessWrong platform. After initial collection and cleaning, the data is organized into three distinct datasets with specific volume and time period requirements:

\begin{itemize}
    \item \textbf{\texttt{lesswrong5}}
    \begin{itemize}
        \item \textit{Target Volume:} 150 articles per author.
        \item \textit{Time Period:} Exactly 5 articles published in or after 2025 ($\ge 2025$), and up to 145 articles published in or before 2024 ($\le 2024$).
    \end{itemize}
    
    \item \textbf{\texttt{cleaned\_10}}
    \begin{itemize}
        \item \textit{Target Volume:} Flexible total, but requires a strict minimum threshold.
        \item \textit{Time Period:} A minimum of 20 old articles ($\le 2022$) AND a minimum of 20 new articles ($\ge 2024$). Articles from 2023 are deliberately omitted.
    \end{itemize}

    \item \textbf{\texttt{cleaned\_35}}
    \begin{itemize}
        \item \textit{Target Volume:} Between 30 and 50 articles per author.
        \item \textit{Time Period:} No specific year constraints.
    \end{itemize}
\end{itemize}

\section{Methodology}

\subsection{Data Crawling}
The data collection was automated using customized Python scripts interfacing directly with LessWrong's official graphql api. To set up the environment, the required dependencies can be installed simultaneously by running the following command in the terminal:

\begin{center}
\begin{minipage}{0.55\textwidth}
\begin{lstlisting}[language=bash]
pip install requests beautifulsoup4
\end{lstlisting}
\end{minipage}
\end{center}

\begin{itemize}
    \item Running \textbf{crawl\_lesswrong5.py} populates the dataset enforcing the strict 5/145 temporal split.
    \item Running \textbf{crawl\_lesswrong10.py} applies the concurrent $\ge 2024$ and $\le 2022$ constraint tracking.
    \item Running \textbf{crawl\_lesswrong35.py} collects the general baseline data without strict time constraints.
\end{itemize}

The crawling infrastructure relies on essential Python libraries:
\begin{itemize}
    \item \textbf{\texttt{requests}:} The core HTTP library used to send POST requests containing JSON-formatted GraphQL queries.
    \item \textbf{\texttt{BeautifulSoup}:} Used to parse the raw \texttt{htmlBody} to strip out structural noise (e.g., \texttt{<img>}, \texttt{<figure>}, \texttt{<script>}) and unwrap hyperlinks to extract pure text.
    \item \textbf{\texttt{re}:} Used to enforce the minimum word count accurately. The pattern \texttt{\textbackslash b[\textbackslash w'-]+\textbackslash b} ensures words with apostrophes (e.g., "don't") and hyphens are counted correctly as single tokens.
\end{itemize}
